# Reglas de Trabajo para Agentes de IA

Este archivo contiene las directrices obligatorias para cualquier IA que trabaje en el proyecto APS. Estas reglas deben ser respetadas en cada sesión para garantizar una colaboración efectiva con el usuario.

## 1. Consulta Previa Obligatoria (CRÍTICO)
No inicies ningún desarrollo, modificación de archivos o refactorización de código sin antes:
1.  **Analizar el requerimiento** del usuario.
2.  **Presentar una propuesta técnica o plan de acción** detallado.
3.  **Obtener la aprobación explícita (el "OK")** del usuario.

*Nota: Ante dudas o consultas teóricas, responde de forma explicativa sin realizar cambios proactivos en el código.*

## 2. Explicación de Arquitectura
Antes de implementar soluciones complejas (como integraciones API, cambios en la base de datos o lógica de algoritmos), presenta siempre diferentes niveles de solución:
- **Solución Simple:** Rápida pero quizás con limitaciones.
- **Solución Recomendada:** El equilibrio óptimo entre seguridad, mantenimiento y rapidez.
- **Solución Avanzada/Escalable:** Para entornos de alta robustez.

## 3. Coherencia Tecnológica
Respeta y mantén el stack tecnológico actual a menos que se indique lo contrario:
- **Core:** React 19 + Vite.
- **Backend/DB:** Supabase.
- **Estilo:** CSS Moderno / Glassmorphism.
- **Internacionalización:** Servicio propio de i18n con soporte multi-idioma.

## 4. Comunicación
- Sé transparente sobre los riesgos de cada enfoque.
- Pregunta siempre antes de asumir preferencias del usuario si no están claras.
- Si cometes un error proactivo, menciónalo y ofrece revertirlo si es necesario.

## 5. Infraestructura y GitHub (PERSISTENTE)
- **Repositorio MCP:** `fguerrausa-a11y/Logic-APS`
- **Rama Principal:** `main`
- Cualquier backup o "push" debe dirigirse exclusivamente a este repositorio.
- Al iniciar una sesión, verifica el estado de este repositorio si el usuario solicita guardar cambios.
